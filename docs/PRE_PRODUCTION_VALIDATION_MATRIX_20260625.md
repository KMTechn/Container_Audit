## Authority Boundary

- Live `C:\Sync`, production HTTPS endpoint, production DB mutation, scheduled-task apply: BLOCKED until explicit approval.
- 승인된 staging/test HTTPS endpoint, test DB, physical/VM PC list, rollback window, 담당 operator가 제공되면 이 문서의 `FIELD_REQUIRED` / `STAGING_REQUIRED` 항목을 순서대로 실행한다.
- 로컬 자동 검증은 production cutover 승인 증거가 아니다. 로컬 PASS는 현장/staging 검증을 시작할 수 있다는 의미다.
- `direct_sync_relay_install_pack.py --apply`, scheduled task/service 변경, credential 회전/폐기, Syncthing 제거는 별도 운영 승인 없이는 실행하지 않는다.
- `docs/COMPANY_SERVER_READONLY_PRECHECK_20260625.md` 기준으로 현재 확인된 회사 서버는 Syncthing peer `Server`가 연결되어 있고 FQDN HTTPS producer route는 `OPTIONS 200`이지만, 실제 POST/HMAC/nonce/idempotency/receipt 저장은 실행되지 않았고 `/health/ingest`는 `PLAN_C_ROLLOUT_GATE_BLOCKED`로 503이다. 실제 업로드 전 approved HTTPS endpoint, credentials, healthy promotion gate, and direct-sync ops schema readiness가 필요하다.
- `docs/OPERATIONAL_DB_READONLY_AUDIT_20260625.md` 기준으로 운영 DB는 legacy dashboard DB로는 healthy지만 `producer_ingest_receipts`, `producer_ingest_nonces`, `producer_ingest_raw_artifacts`, `source_claim`, `source_claim_history`, `transfer_legacy_projection`, `packaging_set_projection`, `process_state_summary_sources`가 없다. `/health/ingest`의 common projection `schema_ready=true`와 별개로 direct-sync ops status는 `BLOCKED`다. 운영 DB 쓰기, schema migration, 실제 producer upload는 DB 백업/rollback window/변경 승인 전까지 금지다.
